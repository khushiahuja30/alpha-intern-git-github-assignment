Merge Conflict Explanation
<br>
A merge conflict occurs when two branches modify the same file differently and Git cannot automatically merge the changes. The conflict was resolved manually and committed.
<br>
Difference Between Fetch and Pull
<br>
Git Fetch :- Downloads changes from a remote repository without merging them.
<br>
Git Pull :- Downloads changes and automatically merges them into the current branch.
